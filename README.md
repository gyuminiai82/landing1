# 🎥 Visionary Projector - Interactive Landing Page

안녕하세요! 가상의 프리미엄 프로젝터 브랜드 **"Visionary"**를 주제로 만들어본 인터랙티브 랜딩 페이지입니다.

"어떻게 하면 프로젝터가 주는 시네마틱한 웅장함을 웹 브라우저에서도 100% 느낄 수 있을까?"라는 고민에서 출발했습니다. 단순히 제품 스펙을 딱딱하게 나열하는 것을 넘어, 사용자가 스크롤을 내릴 때마다 마치 한 편의 영화 예고편을 보는 듯한 몰입감을 주고자 노력했습니다.

## ✨ 제가 가장 신경 쓴 부분들

- **스크롤에 반응하는 시네마틱 애니메이션**: 단순히 위아래로 슥슥 지나가는 웹사이트가 아닙니다. 핀 스크롤(Pinned Scroll) 기법을 사용해서, 스크롤을 내리면 화면이 찌그러지거나(키스톤 보정), 공간의 조명이 켜지거나(IoT 연동), 색감이 변하는(화이트 밸런스) 등 제품의 실제 핵심 기능들을 웹상에서 직접 체험하는 듯한 느낌을 줍니다.
- **몰입감을 더하는 3D & 비주얼 디테일**: 첫 화면(Hero)에서는 마우스 움직임에 반응하는 3D 텍스트와 우주 속으로 빨려 들어가는 듯한 하이퍼스피드 배경을 넣었습니다. 영화관 특유의 묵직한 감성을 살리기 위해 노이즈 필터와 커스텀 커서도 한 땀 한 땀 적용했어요.
- **모바일 환경 최적화**: 데스크탑의 화려한 핀 스크롤도 좋지만, 모바일에서는 억지로 스크롤을 길게 가져가면 꽤나 답답해지잖아요? 그래서 1024px 이하의 모바일 기기에서는 무거운 애니메이션 대신 한눈에 잘 들어오고 직관적인 정적 레이아웃으로 렌더링되도록 분기 처리를 해두었습니다.

## 🛠 사용한 기술들

- **Framework**: React 18 (Vite로 가볍고 빠르게 세팅했어요!)
- **Language**: TypeScript
- **Styling**: Vanilla CSS (CSS Modules) – 불필요한 라이브러리 없이 최대한 순수 CSS로 승부했습니다.
- **Animation**: Framer Motion – 이 프로젝트의 1등 공신입니다. 역동적인 움직임을 모두 담당했습니다.
- **Icons**: Lucide React
- **Deployment**: GitHub Pages

## 🚀 한번 실행해 보실래요?

코드를 클론 받으신 후 아래 명령어만 입력하시면 바로 구경하실 수 있습니다.

```bash
# 1. 의존성 설치
npm install

# 2. 로컬 서버 실행 (구경하기!)
npm run dev
```

(참고: 배포 시에는 `npm run deploy` 명령어를 사용해 GitHub Pages에 올려두고 있습니다.)

## 💦 뼈아팠던 삽질 기록 (Troubleshooting)

개발하면서 겪었던 크고 작은 트러블슈팅 경험들을 잊지 않기 위해 남겨봅니다. 나중에 비슷한 문제를 겪을 누군가에게 도움이 되길 바랍니다.

### 1. "포지션 스티키(position: sticky)가 갑자기 안 먹혀요..."
- **문제**: 모바일 화면에서 가로로 삐져나가는 요소들(오버플로우)을 억지로 숨기려고 최상위 `html, body`에 `overflow-x: hidden`을 주었습니다. 그랬더니 데스크탑 화면에서 잘만 되던 `position: sticky` 핀 스크롤이 완전히 먹통이 되어버렸습니다.
- **해결**: 부모 요소 어딘가에 `overflow: hidden`이 있으면 자식의 `sticky` 계산이 깨지는 브라우저의 고질적인 스펙 문제였습니다. 이를 최신 CSS 문법인 `overflow-x: clip`으로 변경해서 모바일 가로 스크롤 이슈도 잡고, 데스크탑 핀 스크롤도 무사히 살려냈습니다!

### 2. "브라우저 크기를 줄였다 늘리면 화면이 새하얗게 뻗어요"
- **문제**: 데스크탑(스크롤 애니메이션 켜짐) ↔ 모바일(애니메이션 꺼짐) 간에 화면 크기를 마우스로 드래그하며 조절하다 보면, Framer Motion의 동적인 훅(`MotionValue`)과 모바일용 정적 고정값이 렌더링 도중 충돌하면서 리액트 앱 전체가 뻗어버리는 치명적인 에러(`Rendered fewer hooks than expected`)를 만났습니다.
- **해결**: 컴포넌트의 껍데기 HTML이 아니라, 아예 최상위 뼈대인 `App.tsx`에서 뷰포트 상태(모바일/데스크탑)에 따라 각 컴포넌트에 부여하는 `key` 값을 바꿔치기했습니다. 이렇게 하니 모드가 바뀔 때마다 컴포넌트가 완전히 깔끔하게 재부팅(Remount)되면서 애니메이션 엔진의 찌꺼기 값들이 리셋되어 에러가 완벽하게 사라졌습니다.
